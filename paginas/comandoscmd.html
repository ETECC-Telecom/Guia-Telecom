Comandos CMD para Analise de Rede

Segue abaixo uma lista de comandos CMD úteis para análise e solução de problemas de rede:

Para Teste e Análise de Conexão

IPCONFIG

O comando ipconfig é usado para obter informações sobre a configuração de rede em sistemas Windows.

Para executá-lo, siga estas etapas:

  1. Pressione Win + R para abrir a caixa de diálogo Executar.
  2. Digite cmd e pressione Enter para abrir o Prompt de Comando.
  3. No Prompt de Comando, digite ipconfig e pressione Enter.

Esse comando fornecerá informações detalhadas sobre a configuração de rede, incluindo endereços IP, máscara de sub-rede, gateway padrão e muito mais.

ipconfig /all (Subcomando Detalhado)

Embora você já tenha o ipconfig, usar o modificador /all é crucial pois ele exibe informações de configuração de rede muito mais detalhadas para todos os adaptadores, ajudando a verificar se o cliente está pegando o endereço IP, Máscara, Gateway e DNS corretos do seu equipamento (roteador/ONU) ou do servidor DHCP. Algumas informações incluem:

  • Endereço Físico (MAC Address).
  • Se o DHCP está ativado.
  • Servidores DNS e WINS configurados.
  • Tempo de concessão (lease time) do DHCP.

PING

O comando ping é usado para testar a conectividade entre duas máquinas.

Para usá-lo:

  1. Abra o Prompt de Comando (como explicado anteriormente).
  2. Digite ping seguido de um endereço IP ou FQDN (Fully Qualified Domain Name). Por exemplo: C:\> ping www.unicesumar.edu.br
  3. Observe as respostas, latência e perda de pacotes. Por padrão, ocorrem apenas 4 envios de pacote, o que é suficiente para esta tarefa.

Subcomandos:

  • -t: Continua enviando pacotes até ser interrompido.
  • -a: Resolve endereços para nomes de host.
  • -n count: Define o número de solicitações de echo ICMP a serem enviadas.
  • -l size: Define o tamanho do pacote a ser enviado.
Lembre-se de que o número de pacotes padrão é 4. Para especificar 150 pacotes, adicione a opção -n 150 no Windows ou -c 150 no Linux. Por exemplo: ping -n 150 www.google.com (Windows) ou ping -c 150 www.google.com (Linux).

TRACERT (ou TRACEROUTE)

O comando tracert (ou traceroute) é usado para rastrear a rota entre sua máquina e um destino.

Para usá-lo:

  • Abra o Prompt de Comando (como explicado anteriormente).
  • Digite tracert seguido de um endereço IP ou FQDN. Por exemplo: C:\> tracert www.unicesumar.edu.br.
  • Analise as informações fornecidas, identificando os saltos intermediários (hops) na rota.
  • Aguarde, pois o comando pode demorar, mas ele tem um limite de até 30 saltos antes de desistir.

Observação:

Quando o caminho for traçado, alguns casos teremos um retorno em alguns dos caminhos com o nome “esgotado o tempo limite do pedido” que indica que possivelmente o roteador/servidor são bloqueados de receber esse tipo de chamado. Mas isso não indica um erro geral no teste

pathping (Combina ping e tracert)

É uma ferramenta de diagnóstico de rede mais avançada que combina as funcionalidades de ping e tracert. Ela envia pacotes para cada salto (hop) no caminho e, em seguida, calcula a perda de pacotes e a latência em cada um. Leva algum tempo para ser concluído (pode levar alguns minutos, dependendo do número de saltos).

Como usar: pathping www.google.com ou pathping [endereço IP]

Excelente para identificar exatamente onde na rota a perda de pacotes ou o aumento da latência está ocorrendo, ajudando a diferenciar se o problema é na rede interna do cliente, no seu equipamento ou mais adiante na rede do provedor.

nslookup (Consulta de DNS)

É essencial para diagnosticar problemas de resolução de nomes. Ele consulta servidores DNS para obter o endereço IP associado a um nome de domínio.

Como usar:

  • nslookup www.google.com (Verifica a resolução usando o DNS configurado no PC).
  • nslookup www.google.com 8.8.8.8 (Verifica a resolução usando um servidor DNS específico, como o do Google, para contornar temporariamente o DNS do cliente/provedor).

Se o cliente consegue fazer ping para um endereço IP, mas não para um nome de domínio, o problema está quase sempre relacionado à configuração ou falha do Servidor DNS.

Address:  177.185.40.19

Não é resposta autoritativa:
Nome:    www.google.com
Addresses:  2800:3f0:4001:847::2004
          172.217.30.100

netstat -an (Conexões Ativas e Portas)

Útil para ver o estado das conexões do cliente e identificar possíveis problemas de software. Ele exibe as conexões de rede ativas (TCP e UDP), as portas em que o computador está escutando (listening) e o estado das conexões.

Como usar: netstat -an

Ajuda a verificar se o computador do cliente está fazendo ou tentando fazer conexões. O modificador -a (all) mostra todas as conexões e o -n (numeric) mostra endereços e portas em formato numérico (mais rápido).

Para Diagnóstico de Hardware e Link

Teste de Loopback (ping 127.0.0.1)

Um teste fundamental e rápido. Testa o stack (pilha) TCP/IP interno do computador, sem envolver o adaptador de rede ou a rede externa.

Como usar: ping 127.0.0.1 ou ping localhost

Se este ping falhar, o problema não é na sua conexão, mas sim no sistema operacional ou na pilha de rede do cliente, indicando a necessidade de reparo no software do PC.

Teste para o Gateway Padrão (ping [Endereço do Gateway])

O primeiro passo lógico após o ping loopback. Testa a conectividade entre o computador do cliente e o seu roteador/ONU (o primeiro salto).

Como usar: Obtenha o endereço do gateway (ex: 192.168.1.1) via ipconfig e use ping 192.168.1.1 (ou o endereço real).

Uma falha aqui pode indicar problema no cabo, na placa de rede, ou no próprio roteador/ONU. Uma alta latência (acima de 1-2 ms) ou perda de pacotes neste ponto pode indicar problemas de Wi-Fi ou problemas de processamento no roteador local.

Para Manipulação e Reparo

Renovar Concessão de DHCP

Em muitos casos, o cliente pode ter um IP antigo ou incorreto. Libera o endereço IP atual do cliente e solicita um novo ao servidor DHCP.

Como usar:

  1. ipconfig /release
  2. ipconfig /renew

Resolve problemas de conflito de IP e garante que o cliente está usando a configuração de rede mais recente fornecida pelo seu equipamento.

Limpar o Cache de DNS

O cache de DNS pode armazenar informações desatualizadas. Limpa os dados de resolução de nomes armazenados em cache pelo sistema operacional.

Como usar: ipconfig /flushdns

Necessário quando a resolução de nome para um site específico está incorreta, mesmo que o DNS esteja funcionando corretamente (um problema comum após mudanças de host ou IP de servidores).
